### 01. Introduction to Git

##  What is Git?
Git is a **Distributed Version Control System (DVCS)** used to track changes in files and source code during software development.

It helps developers:
✅ Save different versions of code  
✅ Revert back to older versions  
✅ Work in teams without conflicts  
✅ Maintain a history of changes  


##  Why Do We Need Git?

Without Git 
- Files overwrite each other
- No history
- Difficult teamwork
- No backup

With Git 
- Safe and trackable changes
- Team collaboration
- Code backup
- Version history
- Used in DevOps & CI/CD


##  What is Version Control?
Version Control means:

- Tracking changes in code  
- Knowing WHO changed WHAT and WHEN  
- Restoring previous versions anytime  

Example:
v1 → v2 → v3 → v4  
You can go back to v2 anytime 


##  Types of Version Control

###  Centralized (e.g., SVN)
- Single central server
- Requires internet always
- If server crashes → data loss

###  Distributed (Git)
- Every developer has full copy
- Can work offline
- Faster & safer
- No single point of failure

 Git = Distributed = More powerful


##  Git vs GitHub

| Feature       | Git               | GitHub           |

| Type          | Tool/Software     | Cloud Platform   |
| Use           | Version Control   | Host Git Repos   |
| Works         | Offline           | Online           |
| Installed?    | Yes               | No               |
| Example Tools | Git Bash, VS Code | GitLab, Bitbucket|

 Git = Engine  
 GitHub = Parking Garage


##  Real-Time Example

Team building an E-Commerce app:

 Developer A → Adds Login  
 Developer B → Adds Cart  

Without Git → Files overwrite   
With Git → Separate branches,  Safe
