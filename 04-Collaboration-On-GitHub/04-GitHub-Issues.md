### 04-GitHub-Issues

## GitHub Issues

# What are Issues?
Used to track:

- Bugs 
- Tasks 
- Enhancements 
- Questions 

# Why Use Issues?
- Organized work
- Assign tasks
- Track progress
- Comment & discuss

# Steps to Create an Issue

1. Go to GitHub Repo
2. Click Issues
3. Click New Issue
4. Add Title + Description
5. Assign (optional)
6. Submit.

- Close Issue
When task is finished:

Click Close Issue
OR
Close automatically by PR

Fixes #5

(Closes Issue #5 when PR merges)
